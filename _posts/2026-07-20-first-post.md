# 모의해킹(Penetration Testing) 직무 분석

## 1. 선택한 직무 정의

모의해킹은 실제 해커의 공격 방식을 활용하여 기업이나 기관의 시스템에 존재하는 보안 취약점을 찾아내는 직무이다. 허가받은 범위 내에서 웹사이트, 서버, 네트워크, 모바일 애플리케이션 등을 대상으로 침투 테스트를 수행하고, 발견한 취약점을 분석하여 보안 개선 방안을 제시하는 역할을 한다.

주요 업무는 다음과 같다.

- 웹 애플리케이션 취약점 진단
- 서버 및 네트워크 보안 점검
- 침투 테스트 수행
- 취약점 분석 및 보고서 작성
- 보안 개선 방안 제안

---

## 2. 필요 역량 분석

실제 채용공고를 조사한 결과 다음과 같은 공통적인 역량이 요구되었다.

### 조사한 채용공고

- 한전 정보보호 채용공고
- 지엔 모의해킹 및 보안 컨설턴트 채용공고
- 에버스핀 취약점 분석 및 정보보안 모의해킹 컨설턴트 채용공고

### ① 웹 보안 취약점에 대한 이해

대부분의 채용공고에서 웹 보안 취약점에 대한 이해를 요구하였다.

대표적인 내용은 다음과 같다.

- SQL Injection
- XSS(Cross Site Scripting)
- CSRF
- 인증 및 권한 관리 취약점
- OWASP Top 10

특히 OWASP Top 10에 대한 이해를 중요하게 평가하는 경우가 많았다.

---

### ② 운영체제 및 네트워크 지식

모의해킹은 서버와 네트워크를 함께 분석하는 업무이므로 운영체제와 네트워크에 대한 기초 지식이 요구되었다.

공통적으로 요구되는 내용은 다음과 같다.

- Linux 사용 능력
- Windows 운영체제 이해
- TCP/IP
- HTTP와 HTTPS
- 네트워크 기본 구조

---

### ③ 보안 도구 활용 능력

취약점을 진단하기 위한 다양한 보안 도구를 사용할 수 있는 능력이 요구되었다.

대표적인 도구는 다음과 같다.

- Burp Suite
- 웹 프록시 도구
- 취약점 진단 도구
- Python 등을 활용한 간단한 스크립트 작성

일부 채용공고에서는 자동화 스크립트 작성 경험도 우대사항으로 제시하였다.

---

## 3. 이번 여름방학 목표

이번 여름방학에는 모의해킹 분야의 기초 지식을 학습하는 것을 목표로 하였다.

구체적인 목표는 다음과 같다.

- Linux 기본 명령어 익히기
- HTTP, 쿠키, 세션 등 웹 동작 원리 이해하기
- OWASP Top 10 주요 취약점 학습하기
- Burp Suite 기본 사용법 익히기
- 웹 취약점 실습 환경에서 기초적인 모의해킹 실습하기

현재는 실제 모의해킹 프로젝트를 수행할 수준이 아니므로 기본 개념과 도구 사용법을 우선적으로 학습하고자 한다.

---

## 마무리

조사를 통해 모의해킹 직무에서는 웹 보안 지식, 운영체제와 네트워크에 대한 이해, 그리고 보안 도구 활용 능력이 가장 중요한 역량이라는 것을 확인하였다. 이번 여름방학에는 이러한 기초 역량을 학습하여 보안 분야에 대한 이해를 높이고, 앞으로의 학습에 필요한 기반을 마련하고자 한다.
